如何申请韩国全南大学研究生

发表于 2018-11-21 | 更新于 2019-01-27 | 分类于 留学生活

有个学妹有打算申请我目前所在的学校读研的准备（虽然不是同一专业的），所以趁最近有空特意整理一下自己整个申请过程，来供她参考，避免走很多弯路。我是在今年的五月份递交材料申请的，当时还没拿到毕业证，是以预毕业身份申请的。所以这份指南很适合打算以预毕业身份申请的旁友们。

招生日程

全南国立大学，本科和研究生（大学院）一年招生两次，分春季学期（3 月份入学）和秋季学期（9 月份入学）。春季学期是韩国的正学期，即第一个学期，这个和国内是反过来的。如果申请的是本科生的话，感觉申请 3 月份会好一点。研究生的话，就没关系了。因为无论怎样，硕士和博士都是一起上课的，都会很有压力。春季招生和秋季招生又各分为 2 期，如下图所示：

申请网址

准备的过程中，最主要的是熟读招生简章，把需要准备的材料都记下来列个表，办完一项划去一项。下面是申请的时候需要用到的两个地址，这两个网址，招生简章上也有给出：

  • 招生简章获取地址：招生简章地址，2019年招生简章 pdf 参考
    招生简章有韩文，中文和英文 3 个版本。需要下载韩版和中文版，下载韩版是避免中文版中有错误信息和老旧的信息。下载后可以对照两个文件，以韩文版的要求为主。
  • 网上申请地址：网上申请
    这个网站需要自己注册账号，然后找到对应的学校填写申请，并下载申请表。注册的时候，需要缴纳手续费，全南大学的申请手续费是 6 万韩元。这个可以找在韩国的朋友帮忙缴纳费用。

TOPIK 证书

申请的时候，感觉教授比较注重语言能力。同时有韩语和英语的成绩证明是最好的，这样录取也要保障。韩语成绩只认证 Topik 成绩和学校自己语学堂的成绩证明。英文成绩的话可以提交雅思，托福或者托业成绩。第一学期的奖学金主要依据成绩和语言能力给予。目前政策改变了， 需要 Topik 成绩有 6 级（满级）的话，才可以减免第一学期的学费。按成绩给予的奖学金，主要由学校依据排名决定。通常在缴纳学费的时候，就可以知道自己是否取得奖学金，或者可以邮件询问自己的指导教授。

能考取 6 级的话，就尽全力考取吧！

预毕业证明

在国内是没有预毕业证明这种说法的。正常的就是指学籍证明，因为学籍证明上有你毕业的时间，这个就可以证明预毕业。有些学校是提供英文版的学籍证明的，但我们学校是没有的。这个可以和自己专业的教务科老师商量，自己翻译一份英文版，然后和中文版的一起拿去盖章。学信网上也提供英文版的学籍证明，一份要 30 块钱。为了保险起见，我把学校的中英文学籍证明和学信网的英文学籍证明都准备了，然后订在一起组成一份预毕业证明。

学业计划书和自我介绍书

这两份材料也很重要。需要自己认真填写。我当时是自己写了中文版的，然后交给淘宝翻译一份韩文的，之后再自己确认一遍，无误后再复制到学业计划书和自我介绍书的格式文件中（网上填表后，下载的文件就附带这个）。

公证成绩单和户口本

  • 成绩单：
    成绩单公证最好在学校本地的公证处公证，公证成英文证件。这个可以在过完年，开学后就去教务科打印成绩单去公证。

  • 户口本：
    户口本这个要求只针对中国国籍的学生，如果和父母不在同一户口本上还需要提交亲子证明的公证件。这个可以在户籍本地的公证处公证，最好在过年前回家的时候公证好。

其他

还有一些其他比较好准备的材料，我就不多说了，按照招生简章上的仔细准备就可以了。如果是艺术系的话，需要提交补充材料：

此外，依据学科不同，可能还会有安排面试的情况，海外的话，一般都是电话面试。我是工科的，面试的情况比较少。所以，对于面试这一块的经验为 0。但一般来说，全南大学，有面试的情况都是少的。

最后

准备完材料后，需要在指定的时间内寄到学校的协力科。我当时在杭州，用的顺丰快递，寄了 3 天就寄到了。邮寄完材料后就耐心的等待，每天按时查看邮箱，看看是否有教授或者学校联系你。我当时是在回学校，准备毕业典礼的时候，偶然查看邮箱才发现教授给我发的邮件，整整晚了 6 天才回复教授。所幸教授比较好，收留了我这个菜鸡。

韩国每所大学申请的条件都不相同，此篇文章只作为一个参考。希望大家都能够进入自己心仪的学科继续学习。

字符串的中比较运算符有：==，!=，>，<，>=，<= 以及 equals() 方法和 equalsIgnoreCase()方法。这些运算允许在字符串之间进行字母比较。它们对于排序和按字母顺序排序很有用。

>（大于）和 <（小于）运算符按字母顺序在两者不同的第一个字符上比较字符串。因此，有“a”<“b” 和 “1”<“2”，但 “999”>“1000” 因为 9 比 1 大。

所以，需要注意的是，在比较数字字符串时，字符串比较运算符可能会造成混淆，因为数字被视为字符串而不是数字。如果需要比较数字，请将它们转换为整数，浮点数或长整数，而不是字符串。

为了加快 Arduino 的学习，决定周一这一天的上午也用来学习 Arduino。今天还是继续学习蓝牙模块的使用。实现通过蓝牙模块，在手机端语音控制 LED 的功能。这种思路后续再扩宽的话，加上动手能力强的话，完全自己搭建智能家居系统。

本文章主要参考的是这篇：Android 通过蓝牙语音控制 Arduino，非常感谢作者的教程！

设置蓝牙

本人使用的蓝牙模块是 HC-05，由实验室的小伙伴友情提供。因为是他人使用过的，所以得进行初始化设置。

  • 进入 AT 模式：具体参考这篇文章：Arduino使用HC05蓝牙模块与手机连接
  • AT 模式的一些命令：
    进入 AT 模式后，重新设置名称和密码，并将蓝牙设置为从模式，因为这个项目中，手机是主机。

硬件准备

  • Arduino UNO 一个
  • 蓝牙模块 HC-05 一个
  • 红色和黄色 LED 各一个
  • Android 手机（有谷歌套件）
  • 面包板一块
  • 连接线若干

软件准备

  • Arduino IDE
  • 在 Google APP Store 或者其他安卓市场下载 Arduino Voice Control

电路连接

黄色 LED 的连线：长脚 ——>先接个电阻，再连接 13，短脚 ——> GND
红色 LED 的连线：长脚 ——>先接个电阻，再连接 12，短脚 ——> GND
蓝牙的连线：VCC ——> 5V，GND ——> GND，RXD ——> TXD，TXD ——> RXD，这里需要注意蓝牙的 RXD 要与 Arduino 的 TXD 连接，TXD 要与 Arduino 的 RXD 连接，两者是交错连接的。同时在烧录的时候，注意断开 TXD 和 RXD 的连接，要不然烧录的时候会受到干扰，继而会报错。

实现代码

Arduino_Learning/VoiceControlLed

烧录代码

代码编译完没问题后，点击图中所选按钮将其上传至 Arduino UNO 板上。

手机设置

  1. 点击 CONNECT，选择蓝牙，点击连接。

  2. 点击中间的麦克风，进行语音输入

实际结果

  • 语音输入 turn on the yellow：黄色的灯会打开
  • 语音输入 turn on the red：红色的灯会打开
  • 语音输入 blink：两个灯会一起闪烁
  • 语音输入 stop：两个灯会停止闪烁，并保持常亮
  • 语音输入 turn off：两个灯都会熄灭

这个项目的结果最好的显示方式是录视频，但我一个人忙不过来，就放弃了。结果可以参考上面那篇文章的结果。我主要增加了按颜色实现语音控制 LED ，以及 LED 闪烁功能的开启和关闭这些功能。只是代码方面的修改，及语音方面输入的不同。

心得总结

刚开始在看到一篇用 BitVoice 这个工具实现语音控制的文章，但下载了 BitVoice 这个软件后，没有搞懂怎么使用，就放弃了。后来在收藏夹里又突然发现上面这篇文章，就赶紧参考写代码实现了。在实现的过程中，感觉英文发音比较准的话，会比较容易控制；发音不好的话，就会识别不清，得多次输入。

Arduino 语言常用函数说明

发表于 2018-09-25 | 更新于 2019-01-10 | 分类于 Arduino

Arduino 语言

Arduino 是由 C/C++ 混合编写而成。Arduino 语言也继承了 C/C++ 语言的语法。通常所说的 Arduino 语言，是指 Arduino 核心库文件提供的各种应用程序编程接口（Application Programming Interface，简称API）的集合。

代码示例

Arduino 的基本程序由 setup() 和 loop() 组成，这两个函数是必须的。可以发现 Arduino 不像 C/C++ 语言一样包含 main() 函数。但是它也可以有头文件和自定义函数等要素。

时间函数 Time Functions

  • delay()：

    • 描述：将程序暂停给定参数的时间量（以毫秒为单位）
    • 函数原型：delay(ms)
    • 参数：ms - 暂停的毫秒数
    • 返回值：无

  • millis():

    • 描述：返回自Arduino板开始运行当前程序以来的毫秒数。大约50天后，此数字将溢出（回到零）
    • 语法： time = millis()
    • 参数：无
    • 返回值：程序启动后的毫秒数(unsigned long 类型)

      数字 I/O 函数 Digital I/O Functions

  • pinMode()：

    • 描述：将指定的引脚配置为输入或者输出的模式
    • 函数原型：pinModel(pin,mode)
    • 参数：
      • pin：要设置其模式的引脚编号
      • mode：INPUT（输入模式），OUTPUT（输出模式），INPUT_PULLUP（上拉输入模式）
    • 返回值：无

  • digitalRead()：

    • 描述：读取指定数字引脚的值
    • 函数原型：digitalRead(pin)
    • 参数：要读取的数字引脚的编号
    • 返回值：HIGH(1)，LOW(0)

  • digitalWrite()：

    • 描述：将数字 HIGH 或 LOW 的值写入数字引脚
    • 函数原型：digitalWrite(pin,value)
    • 参数：
      • pin：要设置其输出的引脚编号
      • value：HIGH(1)，LOW(0)
    • 返回值：无

      模拟 I/O 函数 Analog I/O Functions

  • analogWrite():

    • 描述：在指定引脚输出指定占空比的 PWM 方波
    • 函数原型：analogWrite(pin,value)
    • 参数：
      • pin：输出引脚
      • value：占空比，介于 0 - 255 之间
    • 返回值：无

  • analogRead():

    • 描述：读取指定引脚的模拟信号值
    • 函数原型：analogRead(pin)
    • 参数：
      • pin：要读取的模拟引脚的编号
    • 返回值：无

在 analogWrite() 和 analogRead() 函数内部，已经完成了引脚的初始化，因此不用在 Setup() 函数中进行初始化操作。

数学函数 Math Functions

  • map():
    • 描述：将数字从一个范围重新映射到另一个范围。
    • 函数原型：map(value, fromLow, fromHigh, toLow, toHigh)
    • 参数：
      • value：要映射的数
      • fromLow：当前值范围的下限
      • fromHigh：当前值范围的下限
      • toLow：目标值范围的下限
      • toHigh：目标值范围的上限
    • 返回值：映射后的值

注意：不会将值限制在该范围内，因为超出范围的值有时是有意义且有用的。如果需要对范围进行限制，则可以在此函数之前或之后使用 constrain() 函数

  • constrain():
    • 描述：将数字约束在一个范围内。
    • 函数原型：constrain(x, a, b)
    • 参数：
      • x：要约束的数字
      • a：范围的下限
      • b：范围的上限
    • 返回值：
      • 如果 x 在（a, b) 范围内：返回 x
      • 如果 x 小于 a：返回 a
      • 如果 x 大于 b：返回 b

通信串口 Serial 的函数

  • begin()：

    • 描述：设置串行数据传输的数据速率，以每秒位数（波特）为单位。
    • 函数原型：Serial.begin(speed) / Serial.begin(speed, config)
    • 参数：
      • speed：比特每秒(波特) - long
      • config：设置数据，奇偶校验和停止位
    • 返回值：无

  • available()：

    • 描述：获取可从串行端口读取的字节数（字符数）。
    • 函数原型： Serial.available()
    • 参数：无
    • 返回值：可读的字节数。

  • read():

    • 描述：读取传入的串行数据。
    • 函数原型：Serial.read()
    • 参数：无
    • 返回值：可用的传入串行数据的第一个字节（如果没有可用的数据，则为-1） - int

  • print():

    • 描述：在串行口以人们可以看懂的 ASSCII 码的形式打印数据。
    • 函数原型：Serial.print(val) / Serial.print(val, format)
    • 参数：
      • val：要打印的值。
      • format：可选值，选择打印出的参数的类型。如：BIN(binary, or base 2), OCT(octal, or base 8), DEC(decimal, or base 10), HEX(hexadecimal, or base 16) 。
    • 返回值：size_t：print() 返回写入的字节数，但读取该数字是可选的。

  • println()：

    • 描述：将数据作为人类可读的ASCII文本打印到串行端口，后跟回车符（ASCII 13或’\ r’）和换行符（ASCII 10或’\ n’）
    • 函数原型：Serial.println(val) / Serial.println(val, format)
    • 参数：同 print()
    • 返回值：size_t：print() 返回写入的字节数，但读取该数字是可选的。

  • parseInt()：

    • 描述：查找传入串行流中的下一个有效整数。
    • 函数原型：Serial.parseInt(char skipChar)
    • 参数：
      • skipChar：用于跳过搜索中指定的字符。例如，用于跳过数千个分频器。
    • 返回值：long：下一个无效的整数

回顾这四年

发表于 2018-06-14 | 更新于 2019-01-10 | 分类于 个人成长

刚下班，回家路上到处都能听到关于世界杯的事情。想了想上一届世界杯是 2014 年，那会儿刚好是高考考完在家躺尸的时候。时间真快，转眼四年就过去了。正好借这个喜庆的日子，回顾一下大学四年发生的一些事情。

关于选专业

为什么会选择电子信息工程这个专业？ 一是因为当时填报志愿的时候，学校会发一本厚厚的填报专业的指南，电子信息工程专业在上面是就业率高且很受推荐的一个专业。于是百度查询，接收到的信息都是这个专业的前景还不错，推荐的人也多。 二是当时对计算机专业的印象就是脑海中的“码农”形象。而且当时家里的一位表叔是从事计算机行业的，听大人说工作很辛苦。最后再加上自己的认知不高，就遵循自己接受到的信息选择了电子信息工程专业。
后来在大一开学的时候，是有一次转专业的机会的，当时有过犹豫要不要转专业。但是浏览了下学校的专业，没有比较感兴趣的专业，对我爸希望我去的师范专业也不是很感冒。于是就毅然决然的留在了电信专业。现在很感谢自己在那时做的决定，虽然对这个专业不是很热爱，但是在这个专业接收到的信息以及一些数学知识的学习，对于现在的我来说都是很重要的积累。

关于韩国交换生活

大二的时候选择了去韩国交换学习，在韩国呆了差不多一年的时间。一直挺羞于对别人讲这件事的，因为在外人看来，这是一件很不务正业的一件事。在他们眼中你就是去玩的，没学到与专业有关的东西。只有我自己和一同前去的同学知道，这一年对于自己来说是转折的一年。
刚去韩国的时候，连句话都说不完整，在餐馆点餐还得用翻译软件去一个个翻译，再决定吃什么。后来加入了学校提供的语言班，一边学习更系统的韩语课程，一边学习专业课，语言的情况才好转了很多。专业课方面，刚开始也听得懵懵懂懂，后面慢慢的就能跟上节奏，而且在班级里认识了一位学霸，跟着混自习室和图书馆。最后在专业课方面也取得了很满意的成绩。
对于韩国的自习室文化，表示很羡慕。在外面每个街道几乎都会有一家自习室，给想学习的人提供环境，通宵的话还有优惠。韩国人很喜欢在考试前夕熬夜，几乎都是在自习室熬夜复习度过的。曾经在寝室楼下的自习室里，目睹过一位准备考托业考试的韩国学生，几乎吃喝睡都是在自习室了度过的，他的位子上还特意准备了一个加湿器。受过这种熏陶之后，回国之后，越来越喜欢独处，越来越喜欢混自习室和图书馆。
回来之后最大的变化在于对于遇到困难会给予很大的耐心。这是以前一直缺少的，以前常常会特别浮躁。回来之后很多观念也发生了改变。那年正好在韩国经历了 AlphaGo 和李世石的围棋人机大战。从而会对 AI 和 Google 上面有很多改观。在面对计算机科学带来的新发展，也有了认知提升。

关于知识付费

在大三，大四这两年里，是知识付费兴起的时候。在这两年，接触过 Android 开发，英语六级，自动化测试，深度学习等这些方面的付费学习。有些课程荒废了，有些课程坚持学下来了。感触是要勇于尝试多尝试，不要怕试错，这样才能找到自己感兴趣的方向。而且技多不压身，多一项技能，多一个了解的领域，何乐而不为呢。还有在想法的时候，就要马上去实践。虽然技多不压身，但是时不待人。
知识付费有很多坑，这点无可厚非，但我觉得更多的是提供了很多教育平权的机会。让我们能够享受到一些优质的教育资源和内容。

上面这三个方面，是在大学四年这期间的发生的对我来说比较重要的几件事。文笔有限，写不出很好的内容。仅趁兴在此做一个记录，回顾下这短暂的四年。
